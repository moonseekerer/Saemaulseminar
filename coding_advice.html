AI 협업 코딩 성장 일지

코딩 지식을 지탱하는 튼튼한 주춧돌을 세웁니다.
AI를 조수가 아닌 동반자로 삼아, 함께 묻고 직접 치며 나만의 코딩 근력을 키워 나갑니다.

🤝 AI 협업 코딩의 3대 원칙

1

"이유 묻기" (Ask Why)

코드를 그냥 받아쓰기 전에 "왜 이렇게 코딩했어?", "이 스타일 속성은 무슨 의미야?"라고 질문해 보세요. 원리를 이해하는 순간, 다음번 코딩에서 그 원리가 자연스럽게 사용자님의 자산이 됩니다.

2

"직접 치기" (Type Yourself)

명령어와 쉬운 코드는 복사-붙여넣기 대신 직접 키보드로 입력해 보세요. 손가락의 기억(Muscle Memory)이 누적되면 컴퓨터 및 터미널과 대화하는 것이 친숙하고 편해집니다.

3

"피드백 받기" (Get Feedback)

전부를 짜달라고 하지 말고, 약 70%는 힌트를 보며 직접 작성한 뒤 에러를 물어보거나 피드백을 요청해 보세요. 겪어본 시행착오의 개수만큼 실력은 비약적으로 도약합니다.

📚 비전공자 필수 깃허브 치트시트

🚀 핵심 4단계 Git 업로드 프로세스

단계 명령어 설명
1. 상태 조회 git status 어떤 파일들이 수정되었는지 변경 상태 체크
2. 파일 staging git add . 수정된 파일들을 업로드 대기소에 staging
3. 버전 커밋 git commit -m "내용" 어떤 작업을 했는지 라벨링하여 로컬 저장
4. 원격 전송 git push origin main 로컬의 버전을 깃허브 원격 서버로 완전 업로드

💡 *만약 push할 때 거절(Rejected)되면 git pull origin main을 먼저 쳐서 원격 버전을 병합한 후 다시 push하시면 됩니다!

🧱 CSS 레이아웃 핵심 치트시트

↕️ Flexbox (1차원 정렬 - 행/열)

속성 예시 설명
활성화 display: flex; 컨테이너 내 자식 요소들을 정렬 가능하게 만듦
정렬 방향 flex-direction: column; 자식들을 세로(column) 또는 가로(row)로 정렬
메인축 정렬 justify-content: center; 정렬 방향 축 기준 중앙, 여백 분배(space-between)
수직축 정렬 align-items: center; 정렬 방향의 수직축 기준 정렬 (세로 기준 중앙 정렬 등)

🕸️ Grid (2차원 정렬 - 행과 열 동시)

속성 예시 설명
활성화 display: grid; 컨테이너에 바둑판형 정렬 활성화
열 배분 grid-template-columns: 1fr 1fr; 가로를 1:1 두 칸으로 분할 (열 배치 정의)
간격 설정 gap: 1.5rem; 자식 카드들 사이의 간격 규정
열 병합 grid-column: 1 / -1; 해당 요소를 첫 열부터 끝 열까지 전체 너비로 확장
